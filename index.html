How to update this dashboard

Weekly update instructions for both users

Current limitation: This dashboard uses data embedded at build time. To enable live Google Sheets sync, follow the instructions below to connect via Google Sheets API. Both users will then always see current data.
Option A — Google Sheets live sync (recommended)
  1. Open your Google Sheet → Extensions → Apps Script
  2. Paste the provided doGet() Web App script (see below)
  3. Deploy as Web App → Access: "Anyone with the link" — this does NOT make your sheet public, only exposes a read-only JSON endpoint
  4. Copy the deployment URL and paste it into the SHEET_URL variable at the top of this HTML file
  5. The dashboard will auto-fetch fresh data each time it loads
// Paste into Apps Script and deploy as Web App
function doGet(e) {
  const ss = SpreadsheetApp.getActiveSpreadsheet();
  const result = {};
  const sheetNames = [
    'Yeild&Members',
    'Group Totals',
    'Weekly Updates',
    'March Totals',
    // Add new month sheets here as you create them e.g. 'April Totals'
  ];
  sheetNames.forEach(name => {
    const sheet = ss.getSheetByName(name);
    if (sheet) {
      result[name] = sheet.getDataRange().getValues();
    }
  });
  return ContentService
    .createTextOutput(JSON.stringify(result))
    .setMimeType(ContentService.MimeType.JSON);
}
Option B — weekly manual update (current method)
  1. Open the Google Sheet → File → Download → Microsoft Excel (.xlsx)
  2. Send the file to your developer / Claude to regenerate the dashboard HTML
  3. Replace the old HTML file and re-share the link with both users
  4. Update the "Updated" date badge in the nav bar (top right)
Sharing access (both options)
  • Host the HTML file on any private server, intranet, or Google Drive (set share to specific people only)
  • If using Google Drive hosting: right-click → Share → add the 2 email addresses → Viewer access
  • For password protection without a server: consider Netlify (free tier) with password-protected deploy, or share via a private Google Sites page
  • The dashboard requires no backend, database, or login system — it is a single self-contained HTML file
